STEP BY STEP

GitHub Pages 搭建流程

跟着做，不翻车。每一步都标注了关键操作点。

01

注册 GitHub 账号

2分钟

访问 github.com，用邮箱注册。免费账号即可使用 GitHub Pages 功能。建议用户名用英文，后续会体现在默认域名中。

02

创建仓库

1分钟

点击 New Repository，仓库名必须为 你的用户名.github.io，勾选 Add README，设为 Public。

03

上传 HTML 文件

3分钟

在仓库中创建 index.html 文件，写入你的网页代码。也可以直接拖拽上传整个文件夹。支持 HTML/CSS/JS 全部文件类型。

04

开启 Pages 服务

1分钟

进入仓库 Settings → Pages，Source 选择 Deploy from a branch，Branch 选 main，保存。等待 1-2 分钟部署完成。

05

访问你的网站

上线!

打开 https://你的用户名.github.io，你的网站就已经在线了。全球 CDN 加速，国内也能直接访问。

灰产运作模式简析

给周老师的简要分享

所谓"灰产"，本质上是利用信息差和技术门槛，在合规边缘获取利益的模式。和 GitHub Pages 免费建站这个话题相关，常见的套路有：

套路一：免费的东西卖钱

GitHub Pages 本身完全免费，但有人把它包装成"建站服务"，收取几百到几千不等的费用。客户不懂技术，觉得能有个网站就行。

套路二：模板倒卖

从免费模板网站下载模板，稍作修改后当作"原创定制"高价出售。成本低到接近零，利润率极高。

套路三：培训割韭菜

把免费教程包装成"付费课程"，标题党营销："月入过万的建站副业"。实际内容在 YouTube 和 B 站都能免费学到。

套路四：域名和服务捆绑

给你"免费建站"，但要求在他们那里买域名（溢价 3-5 倍）或续费"维护服务"。第一年看似免费，后续年年收割。

核心逻辑就一句话：把公开的免费资源，通过信息不对称，转化为付费产品。知道了就不难防。
